## 필수 보안 설정 1: 이중 인증(2FA) 활성화하기

2FA는 로그인이나 중요한 작업 시 비밀번호 외에 추가적인 인증 수단을 요구하는 기능입니다. OKX에서는 다음과 같은 방법을 제공합니다.

### Google Authenticator (GA) 사용하기 (가장 강력 추천)
1.  OKX 웹사이트 로그인 후, 우측 상단 프로필 아이콘 → **[보안 설정]**으로 이동합니다.
2.  '2단계 인증' 섹션에서 **'Google 인증'** 옆의 [설정] 버튼을 클릭합니다.
3.  화면에 나타난 QR 코드를 스마트폰의 **Google Authenticator 앱**으로 스캔합니다.
  *   *스크린샷 예시: OKX 웹사이트에 표시된 큰 QR 코드와 16자리 백업 키 문자열이 보이는 화면.*
4.  앱에 표시되는 6자리 숫자 인증 코드를 OKX 웹사이트에 입력하여 바인딩을 완료합니다.
> **⚠️ 중요**: QR 코드 아래에 표시되는 **16자리 백업 키(백업 코드)**를 반드시 안전한 곳(예: 비밀번호 관리자, 오프라인으로 기록)에 저장하세요. 스마트폰을 분실했을 때 이 코드로 계정을 복구할 수 있습니다.

### SMS 인증 설정하기 (보조 수단)
GA와 함께 SMS 인증을 추가 설정하면 보안 레이어가 한 겹 더 강화됩니다. '2단계 인증' 섹션에서 **'SMS 인증'**을 설정하고 본인 명의의 휴대폰 번호를 등록하세요.

## 필수 보안 설정 2: 출금 주소 화이트리스트 관리하기

이 기능은 **가장 효과적인 자금 도난 방지 장치**입니다. 미리 등록한 주소로만 출금을 허용하며, 새로운 출금 주소를 추가할 때는 엄격한 인증 절차를 거쳐야 합니다.

### 화이트리스트 설정 방법
1.  [보안 설정] 페이지 내 '출금 주소 관리' 섹션으로 이동합니다.
2.  **[출금 주소 화이트리스트 활성화]** 버튼을 클릭합니다. 처음 설정 시 24시간의 대기 시간이 적용될 수 있습니다.
3.  '주소 추가'를 클릭한 후, 출금을 허용할 외부 지갑 주소(예: MetaMask 지갑 주소, 다른 거래소 입금 주소)를 정확히 입력하고 별칭을 붙입니다(예: "나의 메타마스크", "A거래소 BTC 입금주소").
4.  새로운 주소를 추가할 때마다 이메일/SMS 및 2FA 인증을 완료해야 합니다.
  *   *스크린샷 예시: '주소 추가' 팝업 창과 인증 코드 입력 필드가 보이는 화면.*

**✅ 효과**: 해커가 당신의 계정에 침입하더라도, 미리 등록하지 않은 주소로는 코인을 빼낼 수 없습니다. 새로운 주소 추가를 시도하면 본인에게 인증 요청 알림이 가므로 즉시 대응할 수 있습니다.

## 필수 보안 설정 3: API 키 생성 및 권한 제한

알고리즘 트레이딩, 포트폴리오 추적 앱 등을 사용하기 위해 API 키가 필요할 수 있습니다. **API 키는 절대 '모든 권한'으로 생성해서는 안 됩니다.**

### 안전한 API 키 생성 절차
1.  우측 상단 프로필 아이콘 → **[API 관리]**로 이동합니다.
2.  '새 API 키 생성'을 클릭합니다. API 키의 식별 이름을 지정하세요(예: "3Commas 자동매매 전용").
3.  **권한 설정이 가장 중요합니다.** 대부분의 경우 다음 권한만 체크하면 충분합니다:
    *   ✅ **읽기 전용**: 가장 기본적이고 안전한 권한입니다.
    *   ✅ **스팟/마진 트레이딩**: 알고리즘 트레이딩을 위한 필수 권한입니다.
    *   ❌ **출금 권한은 절대 체크하지 마세요.** 이 권한이 없으면 API 키가 유출되더라도 자금 인출이 불가능합니다.
4.  'IP 주소 접근 제한' 옵션을 활성화하고, API 키를 사용할 자신의 고정 IP 주소를 입력합니다. 이렇게 하면 지정된 IP에서만 해당 API 키를 사용할 수 있습니다.
  *   *스크린샷 예시: 권한 선택 체크박스와 IP 제한 설정 필드가 강조된 API 생성 페이지.*

## 필수 보안 설정 4: 계정 활동 모니터링 및 알림 설정

예기치 않은 로그인이나 시도는 보안 침해의 첫 번째 신호일 수 있습니다. OKX는 이러한 활동을 추적하고 알림을 받을 수 있는 기능을 제공합니다.

### 모니터링 설정 방법
1.  [보안 설정] 페이지에서 '계정 활동' 또는 '로그인 기록' 섹션을 찾습니다.
2.  최근 로그인 활동(시간, 장치, IP 위치)을 정기적으로 확인하세요. 낯선 장치나 지역에서의 로그인 기록이 보이면 즉시 [해당 세션 종료]를 클릭하고 비밀번호를 변경하세요.
3.  [알림 설정]으로 이동하여 다음 알림을 반드시 활성화하세요:
    *   새로운 장치 로그인 시
    *   출금 주소 화이트리스트 추가/출금 시도 시
    *   API 키 생성 시
    *   비밀번호 변경 시
    알림 수단은 이메일과 SMS를 모두 등록하는 것이 좋습니다.

## 자주 묻는 질문 (FAQ)

**Q: Google Authenticator를 분실했거나 휴대폰을 초기화했는데 어떻게 해야 하나요?**
A: 바로 당황하지 마세요. 설정 시 안내했던 **16자리 백업 키**가 있다면, 새로 설치한 Google Authenticator 앱에서 '키 입력' 옵션을 선택하여 해당 코드를 입력하면 기존 계정들을 복구할 수 있습니다. 백업 키도 없다면, OKX 고객지원팀에 문의하여 신분 확인 절차를 거쳐 복구를 요청해야 합니다.

**Q: 출금 화이트리스트를 설정하면 출금이 너무 불편해지지 않나요?**
A: 처음 주소를 등록할 때만 추가 인증이 필요합니다. 일상적으로 사용하는 출금 주소(예: 본인의 하드웨어 지갑 주소)를 미리 등록해두면, 이후에는 해당 주소로 출금 시 추가 인증 없이 빠르게 처리할 수도 있습니다. **편리함보다 안전함을 선택하는 것이 현명한 투자자의 자세입니다.**

**Q: 모든 보안 설정을 완료했는데도 피싱 사이트에 속아 로그인 정보를 입력했다면?**
A: 즉시 OKX 공식 웹사이트에 접속해 비밀번호를 변경하고, [보안 설정]에서 **'모든 세션 로그아웃'** 기능을 실행하세요. 이렇게 하면 도난당한 로그인 정보로 접속한 해커의 세션을 강제로 종료시킬 수 있습니다. 이후 계정 활동을 주의 깊게 모니터링하세요.
